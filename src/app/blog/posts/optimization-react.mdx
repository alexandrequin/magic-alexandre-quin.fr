Une application fluide et rapide améliore l’expérience utilisateur et réduit les coûts de performance. Voici 2 techniques essentielles pour optimiser vos projets React.

## Code Splitting : Charger Uniquement ce qui est Nécessaire

Plutôt que de charger toute l’application dès le début, divisez le code en bundles indépendants. React permet cela grâce à React.lazy et Suspense, ce qui réduit le temps de chargement initial et améliore l’expérience utilisateur.

**Attention : Trop de splits = trop de requêtes réseau. Trouvez le bon équilibre !**

## Memoization : Évitez les Calculs Inutiles

La mémorisation (memoization) empêche le recalcul de valeurs inchangées, améliorant ainsi la réactivité. Utilisez React.memo et useMemo pour éviter les re-rendus inutiles et optimiser les performances.

**Ne sur-utilisez pas useMemo ! Il est utile uniquement pour les calculs lourds et les composants fréquemment re-rendus.**

En combinant ces stratégies, vous obtenez des applications plus rapides, scalables et agréables à utiliser.
